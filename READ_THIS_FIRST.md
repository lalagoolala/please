# 🎯 Google OAuth 오류 해결 - 긴급 안내

## 📌 현재 상황

당신이 본 오류:
```
❌ 400 오류: redirect_uri_mismatch
오류 메시지: "이 앱에서 잘못된 요청이 들어왔습니다"
```

**원인**: Google Cloud Console에 localhost URI가 등록되지 않았습니다.

---

## ⚡ 5분 안에 해결하기

### 1️⃣ Google Cloud 열기
👉 **[Google Cloud Console](https://console.cloud.google.com)** ← 여기 클릭

### 2️⃣ 사용자 인증 정보로 이동
```
왼쪽 메뉴 → APIs 및 서비스 → 사용자 인증 정보
```

### 3️⃣ OAuth 클라이언트 ID 수정
```
"웹 애플리케이션" OAuth 2.0 클라이언트 ID 찾아서 클릭
```

### 4️⃣ 두 섹션에 URL 추가

**섹션 A: 승인된 JavaScript 원본**
```
┌──────────────────────────────┐
│ http://localhost:8000        │
├──────────────────────────────┤
│ http://127.0.0.1:8000        │
└──────────────────────────────┘
```

**섹션 B: 승인된 리디렉션 URI**
```
┌──────────────────────────────┐
│ http://localhost:8000/       │
├──────────────────────────────┤
│ http://127.0.0.1:8000/       │
└──────────────────────────────┘
```

### 5️⃣ 저장하고 테스트
```
[저장] → Ctrl+Shift+Delete → Ctrl+F5 → 완료!
```

---

## 🎉 성공 확인

✅ Google 로그인 버튼 클릭
✅ 오류 없이 Google 계정 선택 화면이 나타남
✅ 계정 선택 후 계정 유형(학생/학부모/관리자) 선택 가능
✅ "가입 완료" 후 자동 로그인

## 🆘 문제 해결

### 문제: "여전히 같은 오류가 나옵니다"
**해결**:
```
1. Ctrl+Shift+Delete (캐시 완전 삭제)
2. 브라우저 완전 재시작
3. Ctrl+F5 (강력 새로고침)
```

### 문제: "어느 포트를 사용해야 하나요?"
**확인**:
```
터미널을 보면:
python -m http.server 8000
                      ↑ 이 숫자가 포트!
```

같은 포트 번호를 Google Console에 추가하세요.

### 문제: "여러 포트를 모두 추가해도 안 됩니다"
**해결**:
```
1. 새로운 Client ID 생성
2. 이번엔 localhost 등록 후 생성
3. index.html의 Client ID 업데이트
```

## 💡 팁

✅ **JavaScript 원본**과 **리디렉션 URI**에 모두 추가해야 합니다!
✅ **둘의 차이**: 
   - 원본: `http://localhost:8000` (슬래시 없음)
   - URI: `http://localhost:8000/` (슬래시 있음)

✅ **여러 포트 추가**: 3000, 5000, 8000 모두 추가해도 좋습니다
