## 📊 데이터 흐름도

### 회원가입 프로세스
```
사용자 입력
    ↓
[검증]
├─ 이메일 중복 확인
├─ 필수 입력 확인
└─ 비밀번호 확인
    ↓
pendingUsers 저장
    ↓
"승인 대기" 상태
    ↓
관리자 승인/거부
    ├─ [승인] → users 저장 & 로그인 가능
    └─ [거부] → 삭제
```

### 로그인 프로세스
```
이메일/아이디 + 비밀번호 입력
    ↓
[검색] users 에서 일치하는 계정 찾기
    ↓
┌─ 계정 없음 → 오류 메시지
├─ 계정 있음 (pending) → "승인 대기 중" 메시지
└─ 계정 있음 (approved) → 로그인 성공
    ↓
currentUser = 해당 사용자 정보
    ↓
localStorage 저장
    ↓
UI 업데이트 (사용자명 표시, 로그아웃 버튼 등)
```

### 회원탈퇴 프로세스
```
"회원탈퇴" 버튼 클릭
    ↓
확인 모달 표시
    ↓
"확인" 버튼 클릭
    ↓
[삭제 처리]
├─ currentUser.id로 users 검색
├─ 해당 사용자 제거
└─ localStorage 업데이트
    ↓
세션 삭제 (currentUser = null)
    ↓
로그아웃 & 홈 화면 이동
```
